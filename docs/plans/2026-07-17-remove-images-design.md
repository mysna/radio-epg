# 이미지 기능 완전 제거 설계

## 목표

라디오 EPG 서비스에서 이미지 수집, 변환, 저장, 관리, 공개 제공 기능과 API 응답의 이미지 필드를 모두 제거한다. 편성 수집과 공개 EPG API만 남기며 R2 의존성도 없앤다.

## 범위

- Python 도메인 모델의 `ImageCandidate`와 `images` 배치 필드, 수집 보고서의 이미지 카운트를 제거한다.
- 어댑터가 upstream 이미지 URL을 읽거나 이미지 후보를 생성하지 않게 한다.
- 이미지 발견, 권리 검사, 다운로드, 변환, 게시 모듈과 전용 테스트를 삭제한다.
- Worker의 이미지 ingest, takedown, public serving 라우트와 R2 binding을 제거한다.
- 채널 및 편성 공개 응답에서 `image_url`, `program_image_url`을 제거한다.
- 신규 D1 마이그레이션으로 entity의 `image_asset_id`, `scrape_runs.image_count`, 이미지 테이블과 인덱스를 제거한다. 기존 이미지 메타데이터는 이 마이그레이션에서 영구 삭제된다.
- 현재 운영 동작을 설명하는 README와 API/DB 문서를 이미지 없는 구조로 갱신하고 `docs/images.md`를 삭제한다. 과거 설계 문서는 변경 이력으로 보존한다.

편성표 자체가 PDF 또는 이미지로만 제공되어 현재 `unsupported`인 방송국의 설명은 이미지 제공 기능이 아니라 향후 편성 입력 형식에 관한 것이므로 유지한다.

## 데이터 흐름

어댑터는 채널, 프로그램, 편성만 `AdapterResult`로 반환한다. Collector는 이 데이터를 `ImportBatch`로 만들고 `/v1/admin/import`에 게시한다. Worker는 D1에 편성을 저장하고 공개 채널/편성/현재 방송 API가 텍스트 메타데이터만 반환한다.

## 마이그레이션과 호환성

기존 배포 DB는 테이블 재작성 방식의 신규 마이그레이션으로 이미지 외래 키 컬럼을 제거한다. 이미지 전용 테이블은 참조 해제 후 삭제한다. API 이미지 필드는 호환용 `null`도 남기지 않으므로 클라이언트는 필드 부재를 처리해야 한다. R2 bucket 자체 삭제는 배포 외부의 파괴적 운영 작업이므로 저장소 코드와 binding만 제거하고 자동 삭제하지 않는다.

## 검증

먼저 공개 응답과 import 계약에 이미지 키가 없다는 테스트, Collector/adapter 결과에 이미지 필드가 없다는 테스트, 신규 마이그레이션 후 이미지 스키마가 사라진다는 테스트를 실패시키고 구현한다. 이후 Python 전체 테스트, Worker 전체 테스트와 타입 검사, 저장소 전체 이미지 기능 참조 검색을 실행한다.
