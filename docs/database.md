# Radio EPG 데이터베이스

## 소유권과 저장 위치

D1은 방송사, 채널, 별칭, 프로그램, 편성, 수집 실행 기록과 이미지 메타데이터의
진실의 원천이다. 수집기는 데이터를 직접 수정하지 않고 인증된 ingestion API를 통해서만
배치를 전송하며, Worker가 제약조건과 쓰기 범위를 검증한다.

이미지 원본과 변형 바이트는 D1 BLOB에 넣지 않는다. D1에는 출처, 권리 상태, 콘텐츠 해시,
크기와 R2 객체 키만 저장하고 실제 바이트는 R2에 저장한다. 이렇게 하면 관계형 조회와
백업 크기를 작게 유지하고 이미지 전송은 객체 스토리지/CDN에 맡길 수 있다.

## 핵심 제약조건

- `channel_aliases`는 `(alias_type, alias_value)`를 고유하게 유지한다.
- `schedule_events`는 `ends_at > starts_at`과 0~1 범위의 신뢰도를 강제한다.
- 모든 하위 레코드는 외래키로 소유 레코드를 참조한다.
- 채널별 현재/다음 조회는 `(channel_id, starts_at)` index를 사용한다.
- source/channel/broadcast-date index는 검증된 import 범위 교체에 사용한다.
- 이미지 변형의 R2 키와 이미지 콘텐츠 해시는 중복될 수 없다.

## 멱등성과 트랜잭션

각 import는 `scrape_runs.idempotency_key`를 고유하게 기록한다. 같은 키의 재전송은 기존
결과를 다시 만들지 않는다. 일정 교체는 검증된 source/channel/broadcast-date 범위 안에서만
수행하며, D1 batch가 실패하면 배치 전체를 롤백해야 한다.

## 삭제와 보존

편성 이벤트는 KST 오늘과 내일의 두 방송일만 보존하고 별도 retention 작업에서 범위 밖
날짜를 삭제한다. 프로그램, 채널, 이미지 메타데이터는 편성 삭제와 함께 제거하지 않는다.
별칭은 채널에 종속되어 채널 삭제 시 함께 제거되지만, 운영 중인 채널은 삭제 대신
`active = 0`으로 비활성화한다.

이미지 삭제 요청은 먼저 asset을 사용 불가로 표시한 뒤 R2 객체를 지우고,
`image_takedowns`에 콘텐츠 해시와 원본 URL을 남겨 같은 파일의 재수집을 차단한다.
